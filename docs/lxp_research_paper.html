LXP: A Prototype for Latent Context Transfer Between Language Models

Research report on architecture, implementation, failure analysis, and the latest same-family raw latent-to-actor decode smoke.

Author: Marlon Moenius Project: Latent Exchange Protocol Models: Qwen3.5-0.8B to Qwen3.5-0.8B Device: Apple Silicon MPS

Abstract

LXP investigates whether language-model agents can communicate through compact continuous latent states instead of exchanging natural-language transcripts. The project started with an ambitious heterogeneous multi-agent goal, then converged on a practical same-family MVP: train a Stage-II adapter that compresses sender hidden states into an actor prefix, validates that the latent carries answer information, and tests whether the actor can decode the final answer directly from the latent handoff. The latest local smoke run reaches 100% raw actor free-decode exact match, 100% answer extraction, and 3/3 unique predicted answers on a three-sample train-overfit smoke. This is not yet broad generalization, but it is a functional end-to-end demonstration of latent handoff instrumentation, training, evaluation gates, and collapse diagnosis.

1. Problem and Insight

Multi-agent language-model systems usually communicate by emitting text. Text is inspectable, but it is also slow, lossy, and constrained by the vocabulary interface. A model may internally represent a useful partial solution as high-dimensional activations, then spend many tokens translating that state into natural language before another model re-embeds the text into a new latent space. LXP asks whether that round-trip can be shortened by passing structured latent states directly.

The central hypothesis is modest but useful: before attempting fully heterogeneous transfer, a same-family latent handoff should be made mechanically reliable, diagnostically observable, and empirically gated. If the actor cannot recover answers from a compatible latent prefix, cross-family alignment has no stable target. The project therefore shifted from broad model-pair ambition to a high-signal MVP that isolates baseline generation, latent semantic content, adapter training, and raw actor decoding.

2. System Architecture

The current implementation separates latent transfer into five surfaces: model-pair compatibility preflight, sender hidden-state extraction, latent compression/alignment, actor-prefix injection, and evaluation diagnostics. The architecture avoids treating a single accuracy number as proof of success. Instead, it asks where the information exists, how it is decoded, and whether the decoded answer is non-degenerate.

Figure 1. LXP architecture used for the final local smoke: a sender hidden-state trajectory is compressed into a short latent prefix, adapted into the actor interface, and evaluated through baseline, probe, semantic readout, and raw decode surfaces.

3. Implementation Contributions

The project produced a working codebase rather than a single notebook. Major improvements included preflight compatibility checks, smoke report hardening, semantic evaluation, local Apple Silicon training, raw actor decode gating, and collapse diagnostics. The most important engineering change was moving away from vague "accuracy is zero" reports and toward separate instrumentation for each possible failure point.

Area Implemented Capability Why It Matters
Model compatibility Preflight reports layer count, cache topology, hidden size, heads, KV heads, and transfer status. Prevents impossible cross-model KV transfers from being mistaken for training failures.
Evaluation rigor Reports now distinguish extraction rate, exact match, unique predicted answers, baseline quality, probe accuracy, and raw decode readiness. Degenerate repeated answers can no longer pass silently as apparent progress.
Training path Mac MPS Stage-II smoke runner with adapter answer NLL, first-token objective, logit steering, and early stopping. Allows local iteration when external GPU capacity is unavailable.
Actor collapse fix Raw decode can stop after steered answer tokens and use latent semantic readout only for answer length control. Prevents untrained tail generation from corrupting otherwise correct latent-to-actor handoffs.
Reporting Static HTML reports and demo pages with charts, diagrams, diagnostics, and limitations. Makes the artifact understandable and visually provable for reviewers.

It uses hybrid_affine alignment, 250 semantic anchors, 8 compressed latent steps, float32 execution on MPS, and a three-sample smoke dataset. The run was intentionally small because the immediate goal was to debug latent-interface correctness before spending cloud compute on broader training.

6. Ablations and Failure Analysis

The project uncovered several failure modes that are useful research results in their own right. First, the initial raw actor path repeatedly produced the same answer token. This meant the latent injection path was not enough by itself; the evaluation had to distinguish latent semantic availability from raw actor decode quality. Second, answer extraction was initially too brittle, so outputs that were semantically usable could be reported as failures. Third, heterogeneous KV transfer was not a training problem: some model pairs were structurally incompatible because their layer counts and cache topologies differed.

Fixed or Improved

  • Baseline actor degeneration was isolated from latent handoff failures.
  • Answer extraction now reports extraction rate separately from exact match.
  • Raw decode readiness requires exact answers and non-degenerate predictions.
  • Untrained tail generation no longer invalidates steered raw answer tokens.
  • Compatibility checks avoid impossible KV-transfer expectations.

Still Open

  • Three-sample smoke is not evidence of broad held-out generalization.
  • Heterogeneous family transfer remains a future work item.
  • Direct latent token decoder is not the winning current path.
  • Multi-seed real benchmark retention has not yet been completed.
  • Large-scale cloud training is still needed for production-level claims.

7. Model-Pair Compatibility Findings

A major result was recognizing that model-pair selection is not a minor hyperparameter. Direct KV transfer is only meaningful when the sender and receiver have compatible cache structures. The project therefore added config-only compatibility preflight before full weight loading.

Pair Predicted Compatibility Interpretation
Qwen3.5-0.8B → EXAONE-4.0-1.2B incompatible Layer-count/cache mismatch; keep as a later cross-family alignment benchmark, not first MVP.
Qwen3.5-2B → Qwen3.5-0.8B predicted compatible Better same-family path for a stronger thinker and cheaper actor once compute is available.
Qwen3-1.7B → Qwen3-0.6B predicted compatible Fallback same-family path with matching cache topology in preflight tests.

8. Discussion

The most important lesson is that latent transfer cannot be evaluated only by final answer accuracy. A system can fail because the sender did not solve the task, because the latent has no answer information, because the adapter loses geometry, because the actor cannot condition on the prefix, because the text decoder collapses, or because the answer parser is brittle. LXP now contains enough instrumentation to separate these cases.

The final local smoke result is best interpreted as an interface proof. It demonstrates that the code can train a latent handoff, verify answer information inside the latent, and produce exact parseable actor text on a small controlled set. The result does not yet prove that latent transfer beats token-context transfer or generalizes across tasks. The next decisive experiment is a locked held-out benchmark comparing token-context handoff, verified token-context handoff, sender-answer text handoff, and generated-context latent handoff under the same manifest.

9. Conclusion

LXP produced a substantial functional artifact: a latent transfer training and evaluation system with compatibility preflight, local MPS training, raw actor decode, semantic probes, failure diagnostics, and visual reports. The strongest empirical outcome is a 100% same-family raw latent-to-actor decode smoke on three examples with 100% answer extraction and non-degenerate predictions. The honest next step is to scale this path to locked held-out multi-seed evaluations and benchmark it directly against pure token-to-token context transfer before re-opening heterogeneous model-family transfer.
